# Концепция Интеграции ИИ: Автономная система управления VPN (AIOps)

## 1. Суть концепции (Как это работает)

Система будет представлять собой отдельный независимый Python-сервис (`ai_manager.py`), который будет работать на сервере в фоновом режиме. Он не будет зависеть от веб-панели, что обеспечит высокую отказоустойчивость.

**Цикл работы (Workflow):**
1.  **Мониторинг (Сенсоры):** Сервис непрерывно читает логи Xray (`access.log`, `error.log`) и следит за системными метриками. Он ищет паттерны, характерные для сканеров ТСПУ/DPI (резкий всплеск ошибок TLS, таймауты соединений).
2.  **Анализ (Сотрудник ИИ):** При обнаружении аномалий, сервис собирает выдержку из логов и отправляет их через API на ваш сайт к "Сотруднику ИИ". Сотрудник анализирует логи и предлагает решение (например: "Провайдер блокирует текущий SNI, нужно сменить SNI на `vk.com` и fingerprint на `qq`").
3.  **Проверка (Судья ИИ):** Предложенное решение и текущий конфиг отправляются к "Судье ИИ". Судья оценивает риски: не сломает ли это конфигурацию, корректен ли JSON. Он выносит вердикт: `APPROVE` или `REJECT`.
4.  **Безопасное исполнение:** Если вердикт `APPROVE`:
    *   Создается резервная копия текущего `config.json`.
    *   Применяются изменения.
    *   Служба Xray перезапускается.
    *   Проводится Healthcheck. Если Xray не поднялся (ошибка конфигурации), происходит мгновенный откат (Rollback) к резервной копии в течение 30 секунд.
5.  **Отчетность (Telegram):** О каждом шаге (обнаружение угрозы, решение Судьи, успешное применение или откат) отправляется уведомление в Telegram.

## 3. План разработки

1.  **Настройка окружения:** Создание `ai_manager.py` в проекте, настройка логирования.
2.  **Модуль Telegram:** Написание класса для отправки сообщений, тестирование отправки.
3.  **Модуль Xray-Бэкап:** Написание функций для бэкапа `config.json`, валидации нового конфига и безопасного рестарта службы с откатом.
4.  **Модуль API ИИ:** Интеграция с вашим API по предоставленным форматам (создание промптов для Сотрудника и Судьи).
5.  **Модуль Мониторинга:** Разработка парсера логов для выявления аномалий.
6.  **Финальная сборка:** Объединение всех модулей, создание systemd-службы `vpn-ai.service` для автономной работы.
