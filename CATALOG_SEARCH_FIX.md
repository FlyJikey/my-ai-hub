# Решение проблемы: ИИ видит только 10 товаров вместо 2652

## Проблема

При запросе в чате с включённой "Базой товаров" ИИ видел только 10 товаров из 2652, хотя все товары были векторизованы.

## Причина

Чат и поиск использовали **жёстко прописанную модель** `text-embedding-3-small`, но пользователь мог векторизовать товары **другой моделью** (например, Nomic Embed 1.5 с размерностью 768 вместо 1536).

При несовпадении размерности векторов:
- Запрос пользователя: вектор 1536 измерений (text-embedding-3-small)
- Товары в базе: вектор 768 измерений (Nomic Embed 1.5)
- Результат: векторный поиск не работает корректно

## Решение

### Что было исправлено:

1. **API `/api/catalog/search`** - теперь принимает параметры:
   - `embeddingModel` - ID модели (например, "text-embedding-3-small")
   - `embeddingProvider` - провайдер (polza/openrouter)

2. **API `/api/catalog/chat`** - аналогично принимает модель и провайдер

3. **Фронтенд чата** - передаёт выбранную модель embedding из контекста

4. **Добавлено логирование** для отладки:
   - `[CATALOG SEARCH] Запрос: "...", модель: ...`
   - `[CATALOG SEARCH] Embedding создан, размерность: ...`
   - `[CATALOG SEARCH] Найдено товаров: ...`

## Как проверить, что проблема решена

### Шаг 1: Проверьте модель векторизации

Откройте консоль браузера (F12) и посмотрите логи при векторизации:
```
[VECTORIZE] Начало векторизации с моделью: text-embedding-3-small (polza)
```

Запомните эту модель!

### Шаг 2: Проверьте поиск в чате

1. Откройте чат
2. Включите "База товаров"
3. Задайте вопрос (например, "покажи все товары")
4. Откройте консоль (F12) и найдите логи:

```
[CATALOG SEARCH] Запрос: "покажи все товары", модель: text-embedding-3-small (polza)
[CATALOG SEARCH] Embedding создан, размерность: 1536
[CATALOG SEARCH] Найдено товаров: 20
```

### Шаг 3: Проверьте результат

ИИ должен показать 20 товаров (или сколько нашёл), а не 10.

## Если проблема осталась

### Вариант 1: Несовпадение моделей

**Проблема:** Вы векторизовали товары одной моделью, а чат использует другую.

**Решение:**
1. Откройте страницу "База товаров"
2. Нажмите "Заменить базу данных" (включите чекбокс)
3. Загрузите файл заново
4. Выберите модель **"OpenAI Embedding Small"** (рекомендуется)
5. Нажмите "Векторизовать ИИ"
6. Дождитесь завершения
7. Попробуйте снова в чате

### Вариант 2: Проблема с SQL функцией

**Проблема:** Функция `match_products` в Supabase настроена на размерность 1536, но вы используете модель с другой размерностью.

**Решение:**

Если вы используете модель с размерностью 768 (Nomic Embed 1.5), нужно создать новую функцию в Supabase:

```sql
create or replace function match_products(
  query_embedding vector(768),  -- Измените на вашу размерность!
  match_count int default 20
)
returns table (
  id bigint,
  sku text,
  name text,
  category text,
  description text,
  price numeric,
  attributes jsonb,
  raw_text text,
  similarity float
)
language sql stable
as $$
  select
    id, sku, name, category, description, price, attributes, raw_text,
    1 - (embedding <=> query_embedding) as similarity
  from products
  where embedding is not null
  order by embedding <=> query_embedding
  limit match_count;
$$;
```

И обновите таблицу:
```sql
alter table products 
alter column embedding type vector(768);  -- Измените на вашу размерность!
```

### Вариант 3: Проверьте логи на ошибки

Откройте консоль (F12) и найдите ошибки:
- `[CATALOG SEARCH] Ошибка RPC match_products: ...`
- `[CATALOG SEARCH] Ошибка создания embedding: ...`

Если есть ошибки - сообщите их для дальнейшей диагностики.

1. **Всегда используйте одну модель** для векторизации и поиска

3. Эта модель будет использоваться для новой векторизации
