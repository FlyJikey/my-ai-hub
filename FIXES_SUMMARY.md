# Исправления и оптимизации (24.03.2026)

## Проблемы, которые были исправлены

### 1. Ошибка 504 при загрузке файла
**Причина:** Timeout на Vercel - функция выполнялась дольше 60 секунд

**Решение:**
- Увеличен `maxDuration` с 60 до 300 секунд (5 минут) в `/api/catalog/upload`
- Увеличен `maxDuration` с 60 до 300 секунд в `/api/catalog/vectorize`

### 2. Неправильная замена базы данных
**Проблема:** При нажатии "Заменить базу данных" файл добавлялся к существующим данным вместо полной замены. Файл с 3000 товарами превращался в 28900.

**Решение:**
- Изменён запрос удаления с `.neq('id', 0)` на `.gte('id', 0)` - теперь удаляются ВСЕ записи
- Добавлена поддержка разных форматов флага replace (строка, boolean, число)
- Добавлено логирование процесса удаления

### 3. Медленная загрузка больших файлов
**Решение:**
- Увеличен размер батча с 500 до 1000 записей за раз
- Увеличена параллельность обработки с 5 до 10 одновременных батчей
- Увеличен размер батча векторизации с 100 до 200 товаров

### 4. Плохая индикация статуса векторизации
**Решение:**
- Добавлена зелёная галочка при успешном завершении векторизации
- Улучшена цветовая индикация (жёлтый при ошибках, зелёный при успехе)
- Добавлено сообщение "Векторизация завершена успешно!"
- Статистика обновляется сразу после завершения

## Технические изменения

### `/api/catalog/upload/route.js`
```javascript
// Было:
export const maxDuration = 60;
const BATCH_SIZE = 500;
const CONCURRENCY = 5;

// Стало:
export const maxDuration = 300;
const BATCH_SIZE = 1000;
const CONCURRENCY = 10;
```

### `/api/catalog/vectorize/route.js`
```javascript
// Было:
export const maxDuration = 60;
const BATCH_SIZE = 100;

// Стало:
export const maxDuration = 300;
const BATCH_SIZE = 200;
```

### Логирование
Добавлено подробное логирование всех этапов:
- `[UPLOAD] Начало загрузки каталога`
- `[UPLOAD] Файл: {name}, размер: {size}MB, режим замены: {replace}`
- `[UPLOAD] Распарсено товаров: {count}`
- `[UPLOAD] Режим замены активирован`
- `[UPLOAD] Создана резервная копия: {count} товаров`
- `[UPLOAD] Удалено записей: {count}`
- `[UPLOAD] Начало загрузки: {total} товаров в {batches} батчах`
- `[UPLOAD] Обработка группы батчей {current}/{total}`
- `[UPLOAD] Прогресс: {processed}/{total} ({percent}%)`
- `[UPLOAD] Успешно загружено: {count} товаров`

## Как тестировать

1. **Загрузка файла с заменой:**
   - Загрузите файл с 3000 товарами
   - Убедитесь, что чекбокс "Заменить существующую базу товаров" отмечен
   - Проверьте в консоли логи процесса
   - Убедитесь, что в базе ровно 3000 товаров (не 28900)

2. **Векторизация:**
   - После загрузки нажмите "Векторизовать ИИ"
   - Дождитесь завершения
   - Проверьте, что появилась зелёная галочка "Векторизация завершена успешно!"
   - Убедитесь, что статус изменился на "База полностью готова к анализу"

3. **Мониторинг логов:**
   - Откройте DevTools (F12) → Console
   - Смотрите логи с префиксом `[UPLOAD]` для отладки

## Примечания

- Максимальное время выполнения 300 секунд работает только на Vercel Pro
- Для локальной разработки ограничений по времени нет
- Все изменения обратно совместимы с существующим кодом
